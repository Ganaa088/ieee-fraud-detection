# IEEE-CIS Fraud Detection

A machine learning project built on the [IEEE-CIS Fraud Detection dataset](https://www.kaggle.com/competitions/ieee-fraud-detection) from Kaggle. The goal was to build a model that can identify fraudulent ecommerce transactions from a dataset of 590,000 real-world payments.

I built this as a portfolio project while learning data analysis — specifically to practice the kind of work involved in fraud detection and transaction analysis roles.

---

## Results

| Metric | Score |
|---|---|
| ROC-AUC | **0.9549** |
| Fraud caught (Recall) | 85.4% |
| False alarm rate | 8.8% |

The Kaggle leaderboard winner on this competition scored ~0.945, so landing at 0.9549 on the validation set without heavy hyperparameter tuning was a good result for a first pass.

## Notebooks

**01 — Exploratory Data Analysis**  
Getting familiar with the dataset before touching the model. Covers fraud rate and class imbalance, transaction amount distributions, fraud patterns by hour of day, product type breakdown, card network analysis, and missing value rates across 434 features.

**02 — Preprocessing & Feature Engineering**  
Cleaning up the raw data so it's ready for modelling. Drops columns with >90% missing values, fills the rest with -999 (a sentinel value that lets the model learn "data was absent"), label encodes categorical columns, and engineers 5 new features including hour of day, log-transformed amount, a round-number flag, and a card-level historical fraud rate.

**03 — Model Training & Evaluation**  
Trains an XGBoost classifier on the processed data. Uses `scale_pos_weight` to handle the class imbalance (only 3.5% of transactions are fraud). Evaluates on a held-out validation set using ROC-AUC, confusion matrix, and a plain-English breakdown of what the numbers actually mean.

## How to Run

**1. Clone the repo**
```bash
git clone https://github.com/yourusername/ieee-fraud-detection.git
cd ieee-fraud-detection
```

**2. Install dependencies**
```bash
pip install -r requirements.txt
```

**3. Download the data**  
Download `train_transaction.csv` and `train_identity.csv` from the [Kaggle competition page](https://www.kaggle.com/competitions/ieee-fraud-detection/data) and place them in the `data/` folder.

**4. Run notebooks in order**  
Open Jupyter and run `01_eda.ipynb` → `02_preprocessing.ipynb` → `03_model.ipynb`.  
Notebook 02 needs to finish before 03 since it generates the processed data files.

## Key Findings from EDA

- Only **3.5%** of transactions are fraudulent — significant class imbalance that needs to be handled explicitly in the model
- Product type **'C'** has a fraud rate of 11.7%, more than 3× the overall average
- Fraud rate spikes during **low-volume hours** (late night), consistent with fraudsters operating when fewer human reviewers are active
- Several `id_*` columns from the identity file are **99% empty** — these were dropped before modelling
- Fraudulent transactions cluster in a **narrower amount range** and are almost absent at very high amounts, suggesting fraudsters stay below manual review thresholds

## Notes

This is a learning project. A few things I'd do differently with more time:

- Proper cross-validation instead of a single train/val split
- More feature engineering — the Kaggle winners built 200+ features
- Hyperparameter tuning with optuna or hyperopt
- The `card1_fraud_rate` feature uses the full training set which technically leaks target information — in production this would be computed on training folds only
